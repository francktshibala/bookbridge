### 2. Preview Template

#### Standardized Preview Structure

**BookBridge Preview Template (50-100 words):**


### 3. Length & Format Synthesis

#### Optimal Length by CEFR Level

**Consensus Recommendations:**

- **A2 Level:** 50-75 words (30-45 seconds)
  - Shorter attention span
  - Need quick information
  - 40% higher engagement vs. longer
  - Working memory: 2 chunks

- **B1 Level:** 75-100 words (45-60 seconds)
  - Optimal balance
  - Can process more information
  - Still prefer concise
  - Working memory: 2-3 chunks

- **B2 Level:** 100-125 words (60-75 seconds)
  - Can handle more detail
  - 100 words still optimal
  - Slightly longer acceptable
  - Working memory: 3 chunks

**Universal Rule:** **Keep under 100 words for maximum engagement and comprehension.**

#### Optimal Format by Book Type

**Short Stories (1,000-5,000 words):**
- **Length:** 50-75 words
- **Format:** Text-only (primary)
- **Focus:** Quick information, curiosity hook

**Medium Stories (5,000-15,000 words):**
- **Length:** 75-100 words
- **Format:** Text-only (primary)
- **Focus:** Theme, length, curiosity hook

**Longer Works (15,000+ words):**
- **Length:** 100-125 words
- **Format:** Text-only (primary)
- **Focus:** Theme, length, achievement promise

**Universal Format:** **Text-only as primary, audio optional as supplement.**

### 6. Implementation Plan

#### How to Generate Previews

**Step-by-Step Generation Process:**

1. **Gather Book Information**
   - Book title and author
   - CEFR level
   - Word count and estimated reading time
   - Genre/theme
   - Key plot points (for hook, not spoilers)

2. **Write Preview (50-100 words)**
   - Start with curiosity hook (1-2 sentences)
   - Add theme/genre mention (1 sentence)
   - Include achievement promise (optional)
   - Mention support features (optional, if relevant)
   - Ensure no spoilers

3. **Add Metadata**
   - CEFR level badge
   - Time/length indicator
   - Genre icon

4. **Validate Quality**
   - Check length (50-100 words)
   - Check for spoilers
   - Check language level (matches book level)
   - Check positive framing
   - Review for clarity and engagement

5. **Test Preview**
   - Read aloud (should take 30-60 seconds)
   - Check for clarity and engagement
   - Verify no spoilers
   - Confirm positive framing

**Generation Template:**

```
[Book Title] by [Author]

Level: [CEFR Level] | Time: [X minutes] | Words: [X words]

[Curiosity Hook: 1-2 sentences, no spoilers]

[Theme/Genre: Brief mention, 1 sentence]

[Achievement Promise: Optional, positive framing]

[Support Features: Optional, if relevant]
```

#### How to Validate Quality

**Validation Process:**

1. **Length Check**
   - Word count: 50-100 words
   - Reading time: 30-60 seconds
   - If too long: Remove unnecessary details
   - If too short: Add essential information

2. **Content Check**
   - Difficulty indicator: Present and clear
   - Length/Time: Present and clear
   - Curiosity hook: Present, no spoilers
   - Theme/Genre: Present and clear
   - Language level: Matches book level

3. **Spoiler Check**
   - No twist endings revealed
   - No major plot points revealed
   - No character deaths revealed
   - No surprise revelations revealed

4. **Framing Check**
   - Positive, achievement-focused language
   - No negative or anxiety-creating language
   - Encouraging and supportive tone

5. **Clarity Check**
   - Simple, accessible language
   - Clear and easy to understand
   - No complex or academic terms

**Validation Checklist:**
- [ ] Length: 50-100 words
- [ ] Difficulty indicator: Clear
- [ ] Length/Time: Clear
- [ ] Curiosity hook: Present, no spoilers
- [ ] Theme/Genre: Present
- [ ] Language level: Matches book level
- [ ] Positive framing: Present
- [ ] No spoilers: Verified
- [ ] No overwhelming context: Verified
- [ ] Clear and engaging: Verified

#### How to Test Effectiveness

**Testing Methodology:**

1. **A/B Testing**
   - Test preview vs. no preview
   - Measure: Click-through rate, completion rate
   - Expected: +15-25% improvement with preview

2. **User Feedback**
   - Survey: "Did preview help you decide to read?"
   - Survey: "Was preview helpful or overwhelming?"
   - Survey: "Did preview contain spoilers?"
   - Expected: 70%+ positive feedback

3. **Engagement Metrics**
   - Click-through rate: "Start Reading" clicks
   - Completion rate: % who finish book
   - Abandonment rate: % who abandon in first 5 minutes
   - Expected: +15-25% improvement with preview

4. **Anxiety Metrics**
   - Survey: "Did preview reduce your anxiety?"
   - Survey: "Did preview build your confidence?"
   - Expected: 60%+ anxiety reduction, 35%+ confidence increase

**Success Criteria:**

- **Click-Through Rate:** 40-50% (with preview) vs. 25-35% (without)
- **Completion Rate:** 70-85% (with preview) vs. 55-70% (without)
- **Abandonment Rate:** -50% in first 5 minutes
- **User Feedback:** 70%+ positive feedback
- **Anxiety Reduction:** 60%+ report reduced anxiety

#### How to Iterate

**Iteration Process:**

1. **Collect Data**
   - Engagement metrics (click-through, completion)
   - User feedback (surveys, comments)
   - Abandonment patterns (where students stop)

2. **Analyze Results**
   - Compare preview vs. no preview
   - Identify what works vs. what doesn't
   - Find patterns in successful previews

3. **Refine Preview**
   - Adjust length if needed
   - Improve curiosity hooks
   - Enhance positive framing
   - Remove any spoilers or overwhelming content

4. **Test Again**
   - A/B test refined preview
   - Measure improvement
   - Continue iterating

**Iteration Guidelines:**

- **If click-through low:** Improve curiosity hook, enhance positive framing
- **If completion low:** Check for spoilers, reduce length if too long
- **If abandonment high:** Improve difficulty match, enhance support features
- **If feedback negative:** Review for spoilers, overwhelming context, negative framing
